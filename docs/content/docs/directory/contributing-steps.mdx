# Contributing Steps to the Directory

Learn how to share your custom workflow steps with the community through the Workflow Directory.

## Overview

[Content coming soon]

Contributing your steps to the Directory helps the entire Workflow community. This guide walks you through the contribution process, from preparation to publication.

## Before You Contribute

[Content coming soon]

Before submitting your step, ensure it meets these criteria:

- Well-documented with examples
- Thoroughly tested
- Follows community guidelines
- Has clear type definitions
- Includes a comprehensive README

## Contribution Process

[Content coming soon]

### Step 1: Prepare Your Step

Make sure your step is ready for publication:

- Run all tests
- Update documentation
- Verify package.json metadata
- Add appropriate license

### Step 2: Submit Your Step

[Submission process coming soon]

```bash
# Submit your step to the Directory
npx workflow-directory submit
```

### Step 3: Review Process

[Review process details coming soon]

Your step will be reviewed by maintainers for:

- Code quality
- Documentation completeness
- Test coverage
- Security considerations

### Step 4: Publication

[Publication details coming soon]

Once approved, your step will be published to the Directory and available for the community to use.

## Contribution Guidelines

[Content coming soon]

### Code Standards

- Follow TypeScript best practices
- Use consistent naming conventions
- Implement proper error handling
- Include JSDoc comments

### Documentation Requirements

- Clear description of functionality
- Usage examples
- API reference
- Known limitations

### Testing Requirements

- Unit tests for core functionality
- Integration tests where applicable
- Edge case coverage

## Maintaining Your Steps

[Content coming soon]

After publication, you're responsible for:

- Responding to issues
- Reviewing pull requests
- Publishing updates
- Maintaining compatibility
